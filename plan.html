Product & Ecosystem Plan — April 2026

The Community Hub for Open Broadcast Graphics

A plan to build ograf.dev and ograf.tools — the definitive community portal and interactive workbench for the EBU OGraf ecosystem. Not a replacement for the official spec. The layer that makes it accessible, testable, and actionable.

01 — Positioning
Three Sites, Three Roles
The official spec site stays authoritative. We build the community layer and the developer workbench. The three form a triangle, not a hierarchy.
ograf.ebu.io
The Standard
Normative specification
Official EBU site. Specification docs, JSON schemas, governance, working group. We link to it — we don't duplicate it.
  • Graphics Definition v1 (stable)
  • Server/Control API (draft, mid-2026)
  • JSON schemas & examples
  • Working group governance
ograf.dev
The Community
Learn, connect, discover, contribute
The editorial and community hub. Where a developer, designer, or broadcaster starts their OGraf journey and returns for reference.
  • Tutorials & getting started guides
  • Ecosystem map & compatibility matrix
  • Migration guides (CasparCG, SPX)
  • Community showcase & blog
  • Discord community
  • Contributor guide
ograf.tools
The Workbench
Validate, preview, convert, test
The interactive tooling hub. Where developers open a second tab to validate, preview, explore schemas, or generate templates.
  • Package validator (manifest + full zip)
  • Live preview sandbox with lifecycle controls
  • GDD schema explorer & builder
  • Starter template generator
  • CasparCG migration scorer
  • CLI & CI/CD tools (future)
02 — Killer Features
Five Features That Make This the Bookmark
Each solves a real gap in the current ecosystem. Together they cover discovery, validation, testing, onboarding, and migration.
01

Instant Package Validation

Validate in 10 seconds — ograf.tools/validate

Paste your .ograf.json, get instant line-level errors, warnings, and spec references. Add a "Validate on ograf.tools" badge to GitHub READMEs. Every developer uses this daily — it becomes muscle memory. No signup, no install.

02

Zero-Install Live Playground

The CodePen of broadcast graphics — ograf.tools/preview

Load any OGraf graphic from URL, zip, or gallery. Full lifecycle controls (play, stop, update, customAction), auto-generated data editor from GDD schema, action log, shareable links. The best demo in any meeting: "open this link and click Play."

03

Interactive Ecosystem Map

Every tool in one view — ograf.dev/ecosystem

Visual, interactive landscape of every tool, renderer, editor, and library in the OGraf world. Click any node for details, GitHub stats, and compatibility info. Replaces visiting 10+ different sites. The first thing a CTO evaluating OGraf will bookmark.

04

15-Minute First Template

Guided getting started — ograf.dev/get-started

Step-by-step code-along from zero to a working OGraf lower third. Embedded editor, live preview at each step, downloadable result. Collapses the current 4–5 site onboarding into one linear path.

05

CasparCG Migration Scorecard

Automated assessment — ograf.tools/convert/casparcg

Upload a CasparCG template, get: complexity score (Easy/Medium/Hard), auto-convertible percentage, manual steps checklist, partially-generated OGraf package. CasparCG has 1000+ stars — this taps the largest potential migration audience.

07 — User Journeys
Three Paths Through the Portal

Developer Building Their First Template

Alex — web developer at a regional broadcaster. Knows HTML/CSS/JS. Never built a broadcast graphic.

  1. Lands on ograf.dev homepage → clicks "Get Started"
  2. Follows 15-min tutorial, clicks "Open in Playground" → ograf.tools/preview
  3. Finishes template, validates at ograf.tools/validate → green checkmark
  4. Explores GDD types at ograf.tools/explore/gdd, builds schema visually
  5. Joins Discord, asks questions, gets answers within hours
First working template in 15 minutes. Production-quality in 2–3 hours.

Designer Migrating from CasparCG

Maria — broadcast designer at a sports network. 40+ CasparCG templates in production.

  1. Reads ograf.dev/docs/migration/casparcg → side-by-side comparison
  2. Uploads complex template to ograf.tools/convert/casparcg → "Medium complexity, 70% auto-convertible"
  3. Downloads partially converted package, tests in ograf.tools/preview
  4. Checks ograf.dev/ecosystem/compatibility for renderer feature support
  5. Reports to team with concrete estimates per template type
Concrete migration plan with time estimates, not a vague "we should look into it."

Broadcaster Evaluating OGraf

Thomas — Head of Technology at a medium European broadcaster. Currently uses Vizrt + CasparCG.

  1. Reads ograf.dev "Why OGraf?" → vendor independence, EBU backing, growing ecosystem
  2. Explores ograf.dev/ecosystem → sees SPX, Loopic, CasparCG all supporting OGraf
  3. Opens ograf.tools/preview → loads example templates, shares URL with team
  4. Checks compatibility matrix → cross-references with infrastructure
  5. Reads showcase → sees broadcasters already using OGraf in production
Enough evidence to write an internal recommendation memo.
08 — Launch Strategy
Go-to-Market Plan
Pre-Launch — 2 weeks before

Seed the Community

Create Discord server. Personally invite 10–15 key people: Johan Nyman (SuperFlyTV), Eyevinn team, StreamShapers, SPX creator Tuomo, Loopic team, CasparCG community leaders. Email EBU working group — position as complementary to ograf.ebu.io.

Pre-Launch

Prepare Content

Write 3 launch articles: "Introducing ograf.dev", "First OGraf Template in 15 Minutes", "The OGraf Ecosystem in 2026: A Complete Map".

Launch Day

Lead with the Tool

"Validate your OGraf package in 10 seconds" is more compelling than "read our docs." The validator is the hook.

Launch Day

Hit Every Channel

CasparCG Forum, ebu/ograf GitHub Discussions, Hacker News ("open standard vs $4.9B proprietary market"), Reddit (r/broadcasting, r/obs), X/Twitter (@EBU_Tech), LinkedIn broadcast groups.

Post-Launch — Weeks 1–4

Build Momentum

One blog post per week. Submit to EBU newsletter. Add "Validate on ograf.tools" badges to ecosystem repos. Target: 500 monthly visitors, 100 Discord members in 3 months.

Post-Launch

IBC 2026

IBC is in September. Present at the open-source meetup (where OGraf debuted in 2025). Prepare ecosystem report and demo reel.
